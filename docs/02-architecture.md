## 2.2 核心数据流

```
用户请求 → 接入层(协议转换) → 复杂度评估 → 路由决策 →
    ├─ 简单任务 → 低成本模型 (直接返回)
    ├─ 中等任务 → 中成本模型 (单模型+验证)
    └─ 复杂任务 → MoA 协作 (多模型并行+聚合)
         │
         ├─ 4 个国产模型并行提案
         ├─ 计算共识度
         ├─ 共识 < 阈值? → 追加互审轮
         ├─ 聚合器综合 → 互审员审查
         └─ 修订输出
         ↓
    模型调用(带 Fallback) → 结果返回 → 可观测性记录
```

## 2.4 关键设计决策

### 2.4.1 为什么用 SQLite 而非 PostgreSQL/MySQL?

- **零依赖**:不用装数据库服务,直接落盘
- **跨平台**:Windows/Linux/macOS 行为一致
- **够用**:单机网关,QPS 几千以内完全没问题
- **可升级**:后续可通过修改 `Storage` 类迁移到 PostgreSQL,接口不变

### 2.4.2 为什么用 FastAPI 而非 Flask?

- **异步原生**:MoA 并行调用需要 async/await
- **自动 OpenAPI 文档**:`/docs` 自动出来
- **Pydantic 校验**:请求/响应类型安全
- **流式响应**:`StreamingResponse` 一行代码

### 2.4.3 为什么参考模型不带 tool schema?(仿 Hermes v0.18)

参考模型只做"思考",不直接行动。它们的输出交给聚合器,
聚合器拿完整的 tool schema 再做实际工具调用。

**好处**:
- 减少参考模型调用成本(很多 provider 对 tool schema 收额外 token)
- 避免某些严格 provider 因为带 tool 而拒绝回答
- 集中决策权在聚合器,行为更可控

### 2.4.4 为什么聚合器可以拿全部 schema?

聚合器是"真正的行动者",它需要知道:
- 工具列表(决定能调什么)
- 完整对话历史(决定上下文)
- 参考模型的输出(决定综合什么)

只有它做最终决策,才保证 MoA 行为可预测。

## 2.5 数据持久化

所有数据存到 `data/config.db`:

| 表 | 用途 |
|---|---|
| `admin_users` | WebUI 管理员账号 |
| `api_keys` | 网关 API Key(带 RPM/每日 token 限额) |
| `model_endpoints` | 模型端点配置(API Key 加密存储) |
| `request_logs` | 所有调用日志(可观测性) |
| `config_overrides` | 数据库里的配置覆盖(热更新) |
| `ratelimit_buckets` | RPM 限流计数 |
| `ratelimit_tokens` | 每日 token 计数 |

API Key 用 `cryptography.Fernet` 加密存储,密钥在 `data/.fernet_key`,
JWT secret 在 `data/.jwt_secret`。
