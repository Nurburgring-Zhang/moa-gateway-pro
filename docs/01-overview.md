# 01 · 方案全景与定位

## 1.1 一句话定位

**MoA Gateway Pro** 是一个**纯 Python、可独立运行**的多模型协作网关,
对外提供 **OpenAI 兼容 API**,对内通过 **MoA (Mixture-of-Agents) 编排引擎**让多个国产/国外大模型
**并行提案、共识聚合、互查互审**,从而在**成本可控**的前提下提供
**超越任何单旗舰模型**的回答质量。

## 1.4 适用场景

| 场景 | 推荐配置 |
|---|---|
| **企业内部 AI 中台** | 接入公司所有的模型 Key,统一对外暴露 |
| **Agent 工具** | Hermes / OpenClaw / QoderWork 接入,享受 MoA 增强 |
| **生产环境 LLM 网关** | 健康检查 + 熔断 + 限流 + 日志,生产级稳定 |
| **多模型协作研究** | 提供 4 preset 可调,完整 MoA 编排过程可见 |
| **个人 / 小团队 AI 工具箱** | 5 分钟跑起来,WebUI 配置简单 |

## 1.5 不适用场景

- 需要 **GPU 集群推理** (本项目是 API 聚合层,不直接做推理)
- 需要 **模型微调 / RLHF** (建议用专门的训练平台)
- 需要 **企业级 SSO / 细粒度 RBAC** (v1.0 仅做管理员 + API Key 两级)

## 1.6 设计原则

1. **零信任**:每个模型都可能挂,内置多级 Fallback
2. **智能路由优先**:不盲目选最贵,根据复杂度动态匹配
3. **生产级可靠性**:健康检查、熔断、重试、超时全覆盖
4. **开箱即用**:5 分钟部署,10 分钟接入任意 Agent
5. **可观测性优先**:所有调用可追溯、可统计、可调优
